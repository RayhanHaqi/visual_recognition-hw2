\section{Literature Review}

\subsection{Architectural Evolution: From DETR to RT-DETRv2}
The emergence of the DEtection TRansformer (DETR) introduced a paradigm shift by utilizing a bipartite matching loss and a transformer-based encoder-decoder to eliminate the need for hand-crafted components like Non-Maximum Suppression (NMS) \cite{carion2020}. As proposed by Carion et al., DETR frames detection as a direct set prediction problem.

Real-Time DETR (RT-DETR) later improved upon this by introducing an efficient hybrid encoder to handle multi-scale features, allowing the model to achieve speeds comparable to YOLO architectures while maintaining transformer accuracy \cite{lv2023}. The latest iteration, \textbf{RT-DETRv2}, further optimizes this baseline by refining the backbone interaction and training stability, which is highly effective for dense detection tasks \cite{lv2024}.

\subsection{Optimization and Regularization Strategies}
Deep transformer training typically benefits from the \textbf{AdamW} optimizer \cite{loshchilov2017}, which improves generalization by decoupling weight decay from the gradient updates. Within the DETR loss function, the \textbf{End-of-Sequence (\texttt{eos\_coef})} coefficient is vital for penalizing "no-object" (background) predictions. Tuning this penalty is essential to manage false positives and accommodate potential labeling noise in complex datasets.

\subsection{Resolution Adaptation}
Analysis of the provided dataset indicates a predominantly widescreen aspect ratio. Consequently, the input resolution is adapted to \textbf{$320 \times 640$} to align with the RT-DETRv2 architecture's requirements (multiples of 32) while preserving the natural geometry of the digits.

\section{Methodology}

\subsection{Preprocessing and Resolution Check}
The initial preprocessing phase involved auditing the dataset resolution. To maintain spatial features without vertical distortion, images were resized to $320 \times 640$ pixels. This rectangular input ensures the model learns the correct aspect ratios of the digits.

\subsection{Data Augmentation}
To improve robustness without shifting bounding box coordinates, pixel-level augmentations were employed:
\begin{itemize}
    \item \textbf{ColorJitter:} Random adjustment of brightness, contrast, and saturation to simulate varying lighting environments \cite{carion2020}.
    \item \textbf{RandomErasing:} Strategically masking image patches to simulate occlusions. This technique forces the network to learn robust, alternative descriptive features rather than relying on a single visual cue \cite{zhong2020}.
\end{itemize}

\subsection{Regularization and Training Scheme}
Advanced regularization was implemented to ensure stable convergence:
\begin{itemize}
    \item \textbf{LR Scheduler:} The \textbf{OneCycleLR} scheduler was used to navigate the optimization landscape efficiently. This approach allows for "super-convergence" by utilizing a high maximum learning rate to prevent the model from getting stuck in sharp local minima \cite{smith2019}.
    \item \textbf{EMA:} \textbf{Exponential Moving Average} of model weights, rooted in Polyak averaging \cite{polyak1992}, was utilized to enhance the stability of final predictions by keeping a moving average of the trained parameters.
\end{itemize}

\subsection{Hyper-parameter Experiments}
Systematic experiments were conducted using a multiple GPU setup, focusing on the following variables:
\begin{itemize}
    \item \textbf{Object Queries ($q$):} Evaluation of $q=1000$, $q=1500$, and the default configuration of RT-DETRv2 $q=300$ to accommodate digit density.
    \item \textbf{Weight Decay (WD):} Comparative testing between $1 \times 10^{-4}$, $5 \times 10^{-4}$ and $1 \times 10^{-3}$ for optimal regularization.
    \item \textbf{Learning Rate (LR):} Tuning the base learning rate between $5 \times 10^{-5}$, $1 \times 10^{-4}$, and $5 \times 10^{-4}$ to balance speed and accuracy.
\end{itemize}
To systematically analyze the results, the experiments were categorized into three primary groups:

\begin{enumerate}
    \item \textbf{Baselines:} These models represent the initial configurations using ($q=1000$) number of Queries with variations in Batch Size (16, 24, and 32) and Weight Decay $1 \times 10^{-4}$, $5 \times 10^{-4}$ and $1 \times 10^{-3}$ to establish a performance benchmark.
    \item \textbf{Query Variants ($q300$ \& $q1500$):} These experiments specifically isolate the impact of the transformer query capacity. A lower bound of $q=300$ and an upper bound of $q=1500$ were tested against the baseline to observe its effect on dense digit detection.
    \item \textbf{Optimized Configuration (Best Model):} This group represents the final refinement based on previous findings, utilizing an optimized base Learning Rate of $1 \times 10^{-4}$, Weight Decay of $5 \times 10^{-4}$ and a Batch Size of 24.
\end{enumerate}

\section{Results and Discussion}

A total of 14 distinct training configurations were systematically evaluated. The experiments bridged the evaluation gap between the local validation subset (Mean Average Precision: mAP@50-95) and the unseen test set hosted on the CodaBench platform.

\textbf{Learning Rate Dynamics:} Tuning the base learning rate proved critical for stable model convergence. A lower learning rate of $5 \times 10^{-5}$ (tested in the $q1500$ variant) resulted in slower convergence and a sub-optimal CodaBench score (0.38). Conversely, higher rates such as $5 \times 10^{-4}$ introduced gradient instability during the final epochs. A base LR of $1 \times 10^{-4}$ provided the optimal balance, allowing the OneCycleLR scheduler to effectively traverse the loss landscape \cite{smith2019}.

\textbf{Optimal Queries:} Reducing the object queries to the default $q=300$ degraded the detection capability for highly dense digit clusters, resulting in a score of 0.39. Over-saturating the model with $q=1500$ yielded no performance gain. Thus, $q=1000$ was established as the optimal capacity, providing enough bounding box predictions without excessive computational overhead.

\textbf{Batch Size and Regularization:} The absolute highest CodaBench score (\textbf{0.41}) was achieved utilizing a Batch Size of 24 and a Weight Decay of $5 \times 10^{-4}$. This configuration provided the optimal synergy between batch gradient estimation and regularization, avoiding the overfitting traps observed when pushing the weight decay to $1 \times 10^{-3}$ alongside larger batch sizes, a dynamic consistent with large-minibatch scaling laws \cite{goyal2017}.

\subsection{The Local vs. Test Set Discrepancy (EOS Penalty)}
An innovative experiment was conducted targeting the End-of-Sequence coefficient (\texttt{eos\_coef}) to suppress False Positives, a technique fundamental to balancing the bipartite matching loss in DETR architectures \cite{carion2020}. By increasing the \texttt{eos\_coef} from its default value to $1 \times 10^{-3}$, the model achieved its highest local validation mAP of \textbf{0.4717}.

However, when submitted to CodaBench, the score experienced a severe degradation, dropping to \textbf{0.37}. This phenomenon strongly indicates a significant domain shift or the presence of incomplete annotations in the unseen test set. While the high \texttt{eos\_coef} successfully eliminated False Positives locally, it made the model overly conservative. In the test set, where ambiguous or poorly labeled digits might exist, this strict penalty suppressed True Positives, ultimately degrading the Recall metric. Consequently, the standard \texttt{eos\_coef} with robust weight decay ($5 \times 10^{-4}$) remained the most generalized solution.

\section{Conclusion}

This project successfully adapted the RT-DETRv2 architecture to address the complexities of the dense Digit Detection Problem. Rather than treating the assignment as a standard object detection task, it was approached as a geometric domain adaptation challenge. The implementation of Rectangular Training ($320 \times 640$) effectively preserved the widescreen aspect ratio of the dataset, preventing severe vertical distortion of the digit features while significantly optimizing computational efficiency.

Through a systematic empirical analysis of 14 distinct configurations, the optimal hyper-parameters for this specific dataset were identified. The highest CodaBench test score of \textbf{0.41} was achieved using a configuration of 1000 object queries ($q$), a Batch Size of 24, a base Learning Rate of $1 \times 10^{-4}$, and a Weight Decay of $5 \times 10^{-4}$.

Furthermore, the rigorous experiments involving the bipartite matching loss background penalty (\texttt{eos\_coef}) yielded critical insights. While aggressively penalizing false positives (\texttt{eos\_coef} $= 1 \times 10^{-3}$) resulted in the highest local validation mAP of \textbf{0.4717}, it caused a severe degradation in the unseen test set score (0.37). This discrepancy highlighted the profound impact of noisy or incomplete annotations in real-world test data, demonstrating that an overly conservative model sacrifices recall, and that a balanced regularization approach ($WD=5 \times 10^{-4}$) is paramount for superior generalization.
